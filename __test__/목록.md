        - 테스트: do_clear() [이전검사]

        - 테스트: do_publish()
            + 검사: 파일 유무
            + 검사: 내용 비교 (스냅샷)
        - 테스트: do_clear() [이후검사]
            + 검사: 초기화 됬는지

> 여기서 테스트 해야할 항목은 테스크의 기본적인 작동이 검사
- 기본 작동이란?
    - 출판, 초기화, 오토템플릿 객체 검사

> 태스트에 대한 테스크는 전처리보다 공통화에 의미가 있지 않을까?
하지만 최대한 단순하게 재작해야 한다. 

```bash
npm test -- --collectCoverage : 전체 현황표   
npm test -- -u                : 스캡샷 업데이트   
npm test -- --verbose         : 세부 출력   
```
